Ошибка rate_limited в DeepSeek (VS Code)

Появляется сообщение "rate_limited — превышен лимит запросов к DeepSeek API". Это означает, что сервис запретил новые запросы из-за слишком частых обращений (обычно 10 запросов в минуту).

Почему это происходит?

DeepSeek API имеет ограничение на количество запросов за минуту. Если расширение отправляет слишком много вызовов подряд (например: автодополнение + сразу несколько подсказок), то сервис возвращает ошибку rate_limited и ответы прекращаются.

✅ Способ 1 — Настройка задержки (быстрое решение)

  • Откройте настройки расширения (Continue / Cline / DeepSeek Copilot).
  • Ищите параметр request delay / throttle / rate limit.
  • Установите задержку 6 секунд между запросами.
  • Отключите автодополнение (inline suggestions) — это уменьшает число запросов в фоне.

🌐 Способ 2 — Использование OpenRouter (рекомендуется)

  • Зарегистрируйтесь на openrouter.ai.
  • Получите API-ключ в профиле.
  • В настройках расширения поменяйте baseUrl на https://openrouter.ai/api/v1.
  • В качестве модели используйте deepseek/deepseek-chat или deepseek/deepseek-r1.
  • OpenRouter даёт около 50 бесплатных запросов в день и более гибкие лимиты.

⚡ Способ 3 — SiliconFlow (быстрый, китайский аналог)

  • Зарегистрируйтесь на siliconflow.cn.
  • Получите API-ключ.
  • Укажите baseUrl: https://api.siliconflow.cn/v1.
  • Модель: deepseek-ai/DeepSeek-V3 или deepseek-ai/DeepSeek-R1.

🖥️ Способ 4 — Локально через Ollama (нет лимитов)

  • Установите Ollama.
  • Запустите: ollama pull deepseek-r1:7b (или 13b, если позволяет GPU).
  • В расширении выберите провайдера Ollama.
  • Адрес: http://localhost:11434 — полностью безлимитно.
  • Минус: требуется мощный компьютер / видеокарта для комфортной работы.

      


      

        Пошаговая инструкция:

        3) Перезапусти VS Code (или перезагрузи окно: Ctrl+Shift+P → Reload Window).
